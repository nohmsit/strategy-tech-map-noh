# 국가 기술관리체계 인터랙티브 웹사이트

이 폴더는 **PPT + 엑셀**을 바탕으로 만든 정적 웹사이트입니다.  
GitHub를 처음 써도 올릴 수 있게, 아래 순서대로 아주 자세히 설명합니다.

---

## 1. 이 사이트에서 되는 것

- 19개 분야 탐색
- 기술 검색
- 5개 법령 기준으로 필터
- 분야별 상세 보기
- 법령별로
  - 어떤 기술이 들어가는지
  - 직접 포함(●)인지
  - 연관/유사(○)인지
  구분해서 표시
- 법령별 색상 구분
- 어두운 배경의 대시보드 스타일 디자인

법령 5개는 아래 기준입니다.

1. 국가전략기술육성법
2. 조세특례-국가전략
3. 조세특례-신성장원천
4. 국가첨단전략산업법
5. 산업기술보호법

---

## 2. 폴더 안 파일 설명

- `index.html` → 웹사이트 본문
- `styles.css` → 색상, 레이아웃, 디자인
- `app.js` → 검색, 필터, 상세보기 기능
- `data.json` → 기술 데이터
- `.nojekyll` → GitHub Pages에서 파일이 안정적으로 보이게 하는 설정

---

## 3. GitHub 가입하기

1. 인터넷 브라우저에서 GitHub를 엽니다.
2. 회원가입을 합니다.
3. 가입이 끝나면 로그인합니다.

---

## 4. 새 저장소(Repository) 만들기

1. GitHub 로그인 후 오른쪽 위 `+` 버튼을 누릅니다.
2. `New repository`를 누릅니다.
3. Repository name에 아래처럼 입력합니다.

```text
strategic-tech-map
```

4. Public 을 선택합니다.
5. `Create repository` 버튼을 누릅니다.

---

## 5. 파일 올리기

### 방법 A. 가장 쉬운 방법
1. 이 ZIP 파일을 컴퓨터에 저장합니다.
2. 압축을 풉니다.
3. GitHub에서 방금 만든 저장소 페이지로 들어갑니다.
4. `Add file` 버튼을 누릅니다.
5. `Upload files`를 누릅니다.
6. 압축 푼 폴더 안에 있는 아래 파일을 **전부 한 번에 끌어다 놓습니다.**

- index.html
- styles.css
- app.js
- data.json
- .nojekyll

7. 아래쪽 `Commit changes` 버튼을 누릅니다.

---

## 6. 웹사이트 공개하기 (GitHub Pages)

1. 저장소 화면 상단에서 `Settings`를 누릅니다.
2. 왼쪽 메뉴에서 `Pages`를 누릅니다.
3. `Build and deployment` 또는 `Source` 항목에서  
   `Deploy from a branch`를 선택합니다.
4. Branch는 `main`을 고릅니다.
5. 폴더는 `/(root)`를 고릅니다.
6. `Save`를 누릅니다.

조금 기다리면 주소가 생깁니다.

예시:

```text
https://내아이디.github.io/strategic-tech-map/
```

---

## 7. 사이트가 안 보일 때

보통 1~3분 정도 걸립니다.  
바로 안 보이면:

1. 페이지를 새로고침합니다.
2. 2~3분 정도 기다립니다.
3. 다시 주소를 눌러봅니다.

---

## 8. 나중에 내용 수정하는 법

기술 데이터만 고치고 싶으면 `data.json`만 바꾸면 됩니다.

### 수정 순서
1. 저장소로 들어갑니다.
2. `data.json` 파일을 클릭합니다.
3. 연필 모양(Edit) 버튼을 누릅니다.
4. 내용을 수정합니다.
5. 아래 `Commit changes`를 누릅니다.

잠시 후 사이트에 반영됩니다.

---

## 9. 매우 쉬운 수정 방법

디자인 수정:
- `styles.css`

문구 수정:
- `index.html`

검색 / 필터 기능 수정:
- `app.js`

데이터 수정:
- `data.json`
